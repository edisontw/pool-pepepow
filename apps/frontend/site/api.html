Public API

The PEPEPOW pool API provides read-only status, network, block, payment, and miner summary data for public integrations.

Base URL

https://pool.pepepow.net/api

Quick external integration

Use the stats endpoint for MiningPoolStats and external pool listing sites. Use the status endpoint only when a parser expects a zpool/YiiMP-style object. Detailed endpoints are also available for dashboards and simple scripts.

Endpoints

GET /api/stats

Purpose: Return a NOMP-like pool stats format for MiningPoolStats and external pool listing sites.

Full URL: https://pool.pepepow.net/api/stats

curl -s https://pool.pepepow.net/api/stats | jq

Expected use: main public stats API for pool listings. Includes global workers and hashrate, algo summary, pool summary, worker summary, block counts, and payment totals where available.

GET /api/status

Purpose: Return a minimal zpool/YiiMP-like status object for compatibility fallback.

Full URL: https://pool.pepepow.net/api/status

curl -s https://pool.pepepow.net/api/status | jq

Expected use: secondary compatibility endpoint only. It does not include fake profitability or rental fields.

GET /api/health

Purpose: Check whether the public API is reachable. This is a health check only, not the main stats API.

Full URL: https://pool.pepepow.net/api/health

curl -fsS https://pool.pepepow.net/api/health

Expected use: uptime checks and simple service monitoring.

GET /api/pool/summary

Purpose: Read public pool status, miner counts, share-rate estimate, and recent pool activity.

Full URL: https://pool.pepepow.net/api/pool/summary

curl -fsS https://pool.pepepow.net/api/pool/summary

Expected use: pool dashboards and status widgets.

GET /api/network/summary

Purpose: Read public network height, difficulty, and sync summary shown by the website.

Full URL: https://pool.pepepow.net/api/network/summary

curl -fsS https://pool.pepepow.net/api/network/summary

Expected use: lightweight network status displays.

GET /api/blocks

Purpose: List recent public block and pool candidate observations.

Full URL: https://pool.pepepow.net/api/blocks

curl -fsS https://pool.pepepow.net/api/blocks

Expected use: recent block tables and community status pages.

GET /api/payments

Purpose: List public payment records after operator review and recording.

Full URL: https://pool.pepepow.net/api/payments

curl -fsS https://pool.pepepow.net/api/payments

Expected use: payment history views and miner-facing transparency tools.

GET /api/accepted-candidates

Purpose: List pool-found candidates that have been matched against public chain observations.

Full URL: https://pool.pepepow.net/api/accepted-candidates

curl -fsS https://pool.pepepow.net/api/accepted-candidates

Expected use: public candidate tracking and pool block discovery views.

GET /api/miner/<wallet>

Purpose: Read public summary data for one miner wallet.

Full URL: https://pool.pepepow.net/api/miner/<wallet>

curl -fsS https://pool.pepepow.net/api/miner/YOUR_PEPEPOW_WALLET

Expected use: miner lookup pages, personal status checks, and simple worker monitoring.

Safety and boundaries

  • These endpoints are read-only and intended for website and community status integrations.
  • Stats responses are derived from existing pool snapshots.
  • They do not expose private node services, wallet services, submit controls, admin controls, or runtime JSONL files.
  • Do not depend on private operational details; only use the public JSON fields your integration needs.
  • Payment data is informational and reflects records visible through the public pool website.
  • Use reasonable polling intervals so community dashboards do not overload the pool website.
